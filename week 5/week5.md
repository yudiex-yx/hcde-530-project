# Week 5 — Competency claim

This week I pulled Pokémon data from **PokeAPI** with `requests` and turned it into a **pandas DataFrame** so each Pokémon was one row and fields like `first_type` and `base_experience` were easy to work with. That’s my **data handling** story: I’m not just dumping JSON—I’m cleaning it into a table I can actually analyze with `head()`, `info()`, `value_counts()`, filters, `groupby()`, and `isnull().sum()` to compare types, find stronger Pokémon in the slice, and check for missing cells before I trust an average. On **code reading**, I spent time on each block and practiced explaining what it does in normal language—like how a `groupby("first_type")["weight"].mean()` line turns a tall table into one number per type—so I’m not copy-pasting pandas I can’t walk someone through. For **visualization**, I added a simple **bar chart** of how many Pokémon in my sample have each first type, with a title and axis labels, because the same pattern jumps out faster in a picture than in a frequency table. The analytical questions I answered—type mix, “who’s high on base_experience,” average weight by type—are what connect the API pull to a short plain-English takeaway. I’m still cautious about reading too much into only twenty Pokémon, but I can now follow the thread from API → DataFrame → chart → what I’d tell a classmate in one minute. Altogether that’s what I mean by **C2** (I can read and explain the pandas I used), **C3** (I can load API data, structure it, and analyze it), and **C5** (I can show the same idea in a simple, labeled figure)—as one flow, not three disconnected checkboxes.
